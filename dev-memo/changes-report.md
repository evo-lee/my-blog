### 2. Nested comments

**Modified**
- `api/routers/comment.ts` — full rewrite of `submit`, `listForPost`, `adminList`:
  - `submit` accepts optional `parentId`; pre-checks parent exists / same post / not nested-once-already; insert; catch `SQLITE_CONSTRAINT_FOREIGNKEY` → tRPC `CONFLICT`.
  - `listForPost` returns top-level approved comments + their approved replies, grouped client-shape `{ ...comment, replies: [...] }`. Top-level newest-first, replies oldest-first.
  - `adminList` self-joins via `alias(comments, "parent")` to expose `parentId` + `parentAuthorName`.
- `src/components/Comments.tsx` — group replies, single inline reply form (`activeReplyId` state), indented reply rendering, `PixelAvatar` integration via `CommentRow` sub-component.
- `src/components/admin/CommentsPanel.tsx` — add blue "↳ reply to X" badge when `c.parentId != null`.
- `src/i18n/translations.ts` — add `comments.reply`, `cancelReply`, `replyTo`, `replySubmit` (EN + ZH).

**New**
- `api/comment.test.ts` — 9 tests:
  - valid reply success
  - non-existent parent → NOT_FOUND
  - depth cap (reply to reply)
  - cross-post parent rejected
  - raw insert with bogus parent_id throws (FK enforcement)
  - cascade delete removes replies
  - replies under unapproved parent hidden from `listForPost`
  - replies sorted ASC under each parent
  - `adminList` exposes parentId

### 5. Dual analytics (independent GA + Umami toggles)

**Modified**
- `api/routers/settings.ts`
  - Import `normalizeAnalytics` from `../lib/analytics`.
  - `update` input schema: replace any single-provider fields with `gaMeasurementId` / `umamiSiteId` / `umamiScriptUrl` (all `z.string().max(N)`).
  - After zod parse, call `normalizeAnalytics(...)` in try/catch; map `ZodError` issues to tRPC `BAD_REQUEST` with first issue message.
  - Merge normalized analytics into the `db.update` set.
- `src/components/admin/SiteSettingsPanel.tsx` — full rewrite:
  - `Form` interface adds `gaEnabled`, `gaMeasurementId`, `umamiEnabled`, `umamiSiteId`, `umamiScriptUrl`.
  - `toForm` derives `gaEnabled = gaId !== ''`, `umamiEnabled = umamiId !== '' || umamiUrl !== ''`.
  - UI: two checkbox-gated groups (GA, Umami). Inputs only render when toggle is on. Saving with toggle off clears the corresponding fields.
  - `FieldDef.required` flag; siteTitle / heroTitleEn / heroTitleZh marked required.
  - `REQUIRED_KEYS` const derived from FIELDS.
  - `handleSubmit` pre-validates required fields via trim; shows `clientError` inline if blank.
  - Required labels render red `*`; `<input required={f.required}>`.
- `src/App.tsx` — import and mount `<AnalyticsLoader />` inside `<SetupGuard>` before `<Header />`.

**New**
- `api/lib/analytics.ts` — `normalizeAnalytics(input)`:
  - GA: blank OK; non-blank must match `^G-[A-Z0-9]{6,}$`.
  - Umami: both fields blank OK; both fields together — id must be UUID, URL must parse + be `https:`.
  - Mismatched pairs (id without url / url without id) throw ZodError.
  - Trims all inputs before validating.
- `src/lib/analyticsLoader.ts` — pure helpers:
  - `buildGooglePayload(id)` — returns payload with src `https://www.googletagmanager.com/gtag/js?id=<encoded>`, inline bootstrap that seeds `window.dataLayer` + `window.gtag` and calls `gtag('config', id, { send_page_view: false })`. Sentinel `__gaInit`.
  - `buildUmamiPayload(siteId, url)` — payload with src = url, `dataset.websiteId = siteId`. Sentinel `__umamiInit`.
  - `buildAnalyticsPayloads(cfg)` — returns array of non-null payloads.
- `src/components/AnalyticsLoader.tsx`:
  - Reads `trpc.settings.get.useQuery()`.
  - Only runs when `import.meta.env.PROD`.
  - Computes signature from sorted sentinel pairs; bails if unchanged across re-renders.
  - On change: remove all `script[data-analytics-loader]` nodes, clear `__gaInit`/`__umamiInit` sentinels, then re-inject each active payload. GA gets inline bootstrap first, then external `<script async>`. Umami uses `<script async defer>` with `data-website-id` attr.
- `api/analytics.test.ts` — 10 tests (all combinations: blank, GA-only, Umami-only, both, malformed GA, half-Umami, non-UUID, http URL, garbage URL, whitespace trim).
- `src/lib/analyticsLoader.test.ts` — 5 tests (GA null/valid, Umami null/valid, composition).
